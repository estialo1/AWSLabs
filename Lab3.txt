Step2  (Create a Virtual Server)
Login into your AWS Console
Create an EC2 instance with
Select your default VPC
With the following  Inbound rule
Type      Source
SSH         Anywhere
HTTP       Anywhere IPv4
Select “Amazon Linux 2023 AMI “ under Application and OS Images

Download pem file into your local machine
Create a Security Group with name “PublicSG” and give it any description you prefer
Scroll down to the section with userdata copy and paste this:

#!/bin/bash
# User data script for Amazon Linux 2023
# Installs Apache and serves a custom HTML page

# Update system packages
dnf update -y

# Install Apache HTTP server
dnf install -y httpd

# Write a custom HTML page
cat > /var/www/html/index.html << 'EOF'
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My EC2 Web Server</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            display: flex;
            justify-content: center;
            align-items: center;
            height: 100vh;
            margin: 0;
            background-color: #f0f4f8;
        }
        .card {
            background: white;
            border-radius: 12px;
            padding: 40px 60px;
            box-shadow: 0 4px 20px rgba(0,0,0,0.1);
            text-align: center;
        }
        h1 { color: #232f3e; }
        p  { color: #555; }
        .badge {
            display: inline-block;
            background: #ff9900;
            color: white;
            padding: 4px 12px;
            border-radius: 20px;
            font-size: 14px;
            margin-top: 8px;
        }
    </style>
</head>
<body>
    <div class="card">
        <h1>Hello from EC2! 🚀</h1>
        <p>This page is served by Apache on Amazon Linux 2023.</p>
        <span class="badge">Running on AWS</span>
    </div>
</body>
</html>
EOF

# Enable httpd to start on reboot, then start it now
systemctl enable httpd
systemctl start httpd

# Open a log entry so you can verify the script ran
echo "User data script completed at $(date)" >> /var/log/user-data.log

Then launch the instance.

## Step 1: Clone the Lab Repository

1. I opened **Visual Studio Code** and launched the integrated terminal.

2. I navigated to the AWSLabs folder located on my Desktop using the command:

```bash
cd ~/Desktop/AWSLabs
```

3. I cloned the lab repository from GitHub using:

```bash
git clone https://github.com/awscloudessentials/intro-to-aws-lab.git
```

4. The repository was successfully cloned to my local machine.

---

## Step 2: Access the Cloned Repository

1. I navigated into the cloned repository using:

```bash
cd intro-to-aws-lab
```

2. I listed the contents of the repository to verify the files using:


* Lab1.txt
* Lab2.txt

# Step 5: Login to AWS Console

1. I opened a web browser and logged into the **Amazon Web Services Management Console**.

2. After logging in, I searched for and opened **Amazon EC2** from the services list.

# Step 6: Launch an EC2 Instance

1. From the EC2 dashboard, I clicked **Launch Instance**.

2. I configured the instance with the following settings:

### Instance Name

```
MyWebServer
```

### Application and OS Image

```
Amazon Linux 2023 AMI
```

### Instance Type

```
t3.micro
```
# Step 7: Create a Key Pair

1. I created a new key pair with the following configuration:

Name:
```
Linux
```

Key Pair Type:
```
RSA
```

Private Key File Format:
.pem

2. The **Linux.pem** file was automatically downloaded to my local machine.

# Step 8: Configure Network Settings

1. I selected the **Default VPC**.

2. I created a new **Security Group** with the following details:

Security Group Name:
```
PublicSG
```

Description:
```
Public access for web server
```

3. I added the following **Inbound Rules**:
| Type | Source        |
| ---- | ------------- |
| SSH  | Anywhere      |
| HTTP | Anywhere IPv4 |

This allows remote login and public web access to the server.

# Step 9: Add User Data Script
1. I scrolled down to the **Advanced Details** section.

2. In the **User Data** field, I pasted a script that automatically installs and starts the Apache web server.

3. The script performs the following actions:
* Updates the system packages
* Installs the Apache HTTP server
* Creates a custom HTML webpage
* Starts the Apache service
* Enables the service to start automatically on reboot

This ensures that the web server runs automatically when the EC2 instance launches.
# Step 10: Launch the Instance

1. After completing the configuration, I clicked **Launch Instance**.

2. The EC2 virtual machine was successfully created and started running.
